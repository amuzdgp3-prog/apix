# Архитектурные паттерны — Apix (Slot Manager)

## Структура монорепо
- `apps/client` — PWA (React 18 + Vite + TypeScript)
- `apps/server` — API (Fastify + TypeScript)
- `packages/shared` — общие Zod-схемы и TypeScript-типы

## Ключевые механики

### Placement (сессия аппарата)
- `location` (точка) — вечная сущность.
- `machine` — физический аппарат с номером на корпусе.
- `machine_placement` — сессия: какой аппарат стоит на точке, с какого по какое время.
- Все обслуживания привязаны к `placement_id`.
- Пересчёт цепочки никогда не пересекает границу placement.

### Игрушки на аппарате
- Двухуровневое наследование: тип аппарата → базовый набор → индивидуальные добавления/удаления.
- Техник вводит, сколько штук **доложил**, а не итоговое количество.

### Пересчёт цепочки обслуживаний
- Выполняется **только на сервере**, в одной транзакции с `SELECT FOR UPDATE`.
- Триггеры: добавление, удаление, вставка задним числом.
- Формулы:
  - new_games = game_counter_current - game_counter_prev - test_games_current
  - revenue = new_games × price_per_game (снапшот)
  - cost_of_toys = Σ (qty × price_snapshot)
  - roi = revenue / prev.cost_of_toys
  - period_days = date_current - date_prev

### Снапшоты цен
- Цена игры и цены игрушек фиксируются на момент обслуживания.
- Изменение справочных цен НЕ пересчитывает старые обслуживания.

### Интервалы обслуживания
- Наследование: район → маршрут → аппарат (min_service_days, max_service_days).
- Превышение max_days → аппарат «забытый».
