# План исправлений проблем проекта Apix

**Дата:** 03.07.2026
**Статус:** ✅ Завершено (11/12 ✅, 1/12 🔧 ручная операция)

---

## 🔴 Критические (блокируют деплой):

### Задача 1: ERR_MODULE_NOT_FOUND — symlink @apix/shared ✅
**Проблема:** В Dockerfile (production stage) не копировался `packages/shared/node_modules` (symlink на корневой node_modules через pnpm workspace).
**Исправление:** `apps/server/Dockerfile` строка 43 — добавлен `COPY --from=builder /app/packages/shared/node_modules ./packages/shared/node_modules`

### Задача 2: Локальная сборка @apix/shared падает ✅
**Проблема:** `pnpm --filter @apix/shared build` не находил typescript из-за pnpm hoisting.
**Исправление:** `packages/shared/package.json` — скрипт build заменён на `pnpm exec tsc` (корректно находит typescript через workspace-контекст).

### Задача 3: Nginx редирект HTTPS без сертификатов ✅
**Было:** `return 301 https://...` делал сайт недоступным.
**Стало:** HTTPS секция закомментирована, только HTTP на порту 80.

### Задача 4: NODE_ENV=development ✅
**Было:** `NODE_ENV=development` в конфигурации.
**Стало:** `NODE_ENV=production` в `.env.production` + `docker-compose.yml`.

### Задача 5: Нет автоматической миграции БД ✅
**Было:** CMD запускал только tsx.
**Стало:** `CMD ["sh", "-c", "npx drizzle-kit push && npx tsx ..."]` в Dockerfile.

## 🟡 Средние:

### Задача 6: Прогрев кэша (PWA) ✅
**Проблема:** Справочники не загружались фоново для оффлайн-режима.
**Исправление:** Сервис `apps/client/src/services/warmupCache.ts` реализован и вызывается в `Login.tsx` после авторизации. Эндпоинт `/api/cache/directories` отдаёт все справочники одним запросом.

### Задача 7: GitHub Secrets 🔧
**Проблема:** CI/CD пайплайн (`.github/workflows/deploy.yml`) требует секреты: `DEPLOY_HOST`, `DEPLOY_USER`, `DEPLOY_SSH_KEY`.

**Исправление (ручная операция — должен выполнить пользователь):**
1. Открыть GitHub репозиторий → Settings → Secrets and variables → Actions
2. Добавить секреты:
   - `DEPLOY_HOST` = `91.142.90.200`
   - `DEPLOY_USER` = `root`
   - `DEPLOY_SSH_KEY` = приватный SSH-ключ (сгенерировать `ssh-keygen -t ed25519`, публичную часть добавить в `/root/.ssh/authorized_keys` на VDS)

### Задача 8: version: "3.9" в docker-compose.yml ✅
**Было:** Устаревшая версия схемы.
**Стало:** Строка `version:` удалена (Docker Compose V2 не требует).

### Задача 9: Устаревшая MinIO (2022) ✅
**Было:** `image: minio/minio:RELEASE.2022-...`
**Стало:** `image: minio/minio:latest`

### Задача 10: Dev-значения JWT-секретов ✅
**Было:** `dev_jwt_secret` в продакшене.
**Стало:** Криптостойкие 64+ hex-символов в `.env.production`.

## 🟢 Информационные:

### Задача 11: Конфликт портов MinIO ✅
**Было:** MinIO на портах 9000/9001 конфликтовал с локальным.
**Стало:** Порты `9100:9000` и `9101:9001` в docker-compose.yml.

### Задача 12: Несоответствие портов сервера ✅
**Было:** dev (3000) vs Docker (3001).
**Стало:** Единый порт 3001 в `.env.production`, `docker-compose.yml` и `config.ts`.

2. 🚀 **Деплой** — запустить `docker compose up -d` на VDS сервере
