# Правила выполнения задач — исполнять неукоснительно

## 1. Правила терминала (PowerShell)
Все правила PowerShell → см. `.clinerules/powershell-mandatory.md` (таблица замен, предотвращение зависаний, проверка TypeScript, стандартные шаблоны).

## 2. Правила работы с файлами
- Запрещено создавать временные JS-скрипты для записи файлов.
- Если файл не помещается в один `write_to_file`:
  - Разбей его на 2-3 логические части.
  - Запиши часть 1 → проверь, что записалась → запиши часть 2.
- При 3 неудачных попытках записи одного файла → остановись и скажи: "Нужна ручная вставка".

## 3. Правила безопасности и остановки
- При 3 неудачных попытках ЛЮБОЙ операции → остановись и запроси помощь.
- При получении ошибки API → не пытайся повторить то же действие, переключись на другое.
- Не читай больше 5 файлов одновременно без явного запроса.

## 4. Правило бэкапов перед изменениями на VDS (SSH)

ПЕРЕД ЛЮБЫМИ SSH-командами, которые изменяют файлы или перезапускают службы на VDS, ОБЯЗАТЕЛЬНО:

### А. Бэкап конфигов (nginx, php, apache, supervisor, systemd, .env)
```bash
ssh root@91.142.90.200 "mkdir -p /backup && tar -czf /backup/config_\$(date +%Y%m%d_%H%M%S).tar.gz /etc/nginx /etc/php* /etc/apache2 /etc/supervisor /etc/systemd/system/*.service /var/www/*/.env 2>/dev/null; echo \$?"
```
Если код возврата != 0 — ОСТАНОВИСЬ и сообщи пользователю.

### Б. Бэкап Docker volumes перед `chmod`, `chown`, удалением файлов или пересозданием контейнеров
```bash
# Посмотреть список volume'ов
ssh root@91.142.90.200 "docker volume ls"

# Для каждого критичного volume (например, *_client_dist, *_db_data) создать архив:
ssh root@91.142.90.200 "docker run --rm -v ИМЯ_VOLUME:/data -v /backup:/backup alpine:latest tar -czf /backup/volume_ИМЯ_VOLUME_\$(date +%Y%m%d_%H%M%S).tar.gz -C /data . 2>&1; echo \$?"
```

### В. Бэкап БД PostgreSQL перед миграциями
```bash
ssh root@91.142.90.200 "docker exec apix-db pg_dump -U apix apix | gzip > /backup/db_\$(date +%Y%m%d_%H%M%S).sql.gz 2>&1; echo \$?"
```

### Г. Что делать при ошибке бэкапа
- Если архивация не удалась (код возврата != 0) — ОСТАНОВИСЬ и сообщи пользователю.
- Не продолжай изменения без успешного бэкапа.

### Д. Откат
- При любой ошибке после изменений — восстановить конфиги/volume'ы из последнего архива в /backup.
- Для volumes: `docker run --rm -v ИМЯ_VOLUME:/data -v /backup:/backup alpine:latest tar -xzf /backup/volume_ИМЯ_VOLUME_ДАТА.tar.gz -C /data`

### Е. Очистка старых бэкапов
- Хранить не больше 10 последних бэкапов каждого типа.
- Раз в месяц (или по запросу) чистить: `ssh root@91.142.90.200 "ls -t /backup/*.tar.gz | tail -n +11 | xargs -r rm"`
