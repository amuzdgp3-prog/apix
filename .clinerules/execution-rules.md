# Правила выполнения задач — исполнять неукоснительно

## 1. Правила терминала (PowerShell)
- Разделитель команд: ТОЛЬКО `;`. Запрещены `&&`, `||`.
- Перенаправление: ТОЛЬКО `*>&1`.
- Переменные: `$env:VAR = "value"`.
- При любой проверке TypeScript вывод писать в файл и затем читать его:
  `cd apps/server; npx tsc --noEmit *>&1 | Out-File -FilePath tsc_output.txt -Encoding UTF8`

## 2. Правила работы с файлами
- Запрещено создавать временные JS-скрипты для записи файлов.
- Если файл не помещается в один `write_to_file`:
  - Разбей его на 2-3 логические части.
  - Запиши часть 1 → проверь, что записалась → запиши часть 2.
- При 3 неудачных попытках записи одного файла → остановись и скажи: "Нужна ручная вставка".

## 3. Правила безопасности и остановки
- При 3 неудачных попытках ЛЮБОЙ операции → остановись и запроси помощь.
- При получении ошибки API → не пытайся повторить то же действие, переключись на другое.
- Не читай больше 5 файлов одновременно без явного запроса.
