# Правила таймаутов для долгих команд

## 1. Максимальное время ожидания
- Любая команда, выполняющаяся дольше **60 секунд**, должна быть завершена.
- Не ждать ответа бесконечно — если терминал не отвечает 60+ секунд, нажать Ctrl+C.

## 2. Что делать при зависании
- Завершить команду (Ctrl+C в терминале).
- Проверить результат через независимую команду (например, `docker ps` вместо ожидания `docker compose up`).
- Если команда критическая (деплой, сборка) — разбить на мелкие шаги и выполнять по одному.

## 3. Правило для SSH-команд на VDS
- SSH-команды не должны выполняться дольше 120 секунд.
- Для долгих операций (docker build, docker compose up) использовать фон:
  ```
  ssh root@91.142.90.200 "nohup sh -c 'docker compose up -d' > /tmp/deploy.log 2>&1 &"
  ```
- После запуска фоновой команды проверить результат через 10-15 секунд отдельным SSH-вызовом.

## 4. Правило для локальных долгих операций
- `tar`, `npm install`, `docker build` — запускать с таймаутом.
- Если команда не укладывается в 60 секунд — сообщить пользователю и предложить альтернативу.

## 5. Запрещённые паттерны
- ❌ `docker compose up -d && docker compose logs -f` — logs -f висит бесконечно
- ❌ `tail -f` без таймаута
- ❌ `watch` без таймаута
- ❌ Любая команда с интерактивным режимом без явного завершения
